# 要件定義書 — 部署内書籍管理アプリ

## インテント分析

| 項目 | 内容 |
|---|---|
| ユーザーリクエスト | 部署内で管理している書籍の管理アプリを作りたい |
| リクエストタイプ | 新規プロジェクト（グリーンフィールド） |
| スコープ | 複数コンポーネント（フロントエンド + バックエンドAPI + データベース） |
| 複雑さ | 標準（CRUD中心 + 認証付き） |

---

## 1. プロジェクト概要

部署内の書籍を一元管理するWebアプリケーション。部署メンバーがログイン後、書籍の検索・閲覧・貸出申請・返却を行える。書籍の登録・編集・削除・貸出管理もすべてのメンバーが実施可能。Dockerコンテナで社内サーバーに展開する。

---

## 2. 機能要件

### 2.1 書籍管理
- **FR-01**: 書籍の登録（タイトル、著者、ISBN、出版社、出版年、カテゴリ、保管場所）
- **FR-02**: 書籍の編集（登録した情報の変更）
- **FR-03**: 書籍の削除（蔵書からの除籍）
- **FR-04**: 書籍の保管場所をマスターデータとして管理（ドロップダウン選択、追加・編集・削除可能）

### 2.2 書籍検索・一覧
- **FR-05**: 書籍一覧の表示（ページネーション対応）
- **FR-06**: キーワード検索（タイトル・著者・ISBNで検索）
- **FR-07**: カテゴリによるフィルタリング
- **FR-08**: 貸出状況の確認（在架 / 貸出中 の表示）

### 2.3 カテゴリ管理
- **FR-09**: カテゴリの作成・編集・削除
- **FR-10**: 書籍へのカテゴリ割り当て

### 2.4 貸出・返却管理
- **FR-11**: 貸出申請（ログイン中のユーザー名で自動記録）
- **FR-12**: 返却処理
- **FR-13**: 現在の貸出状況一覧の確認（ソート機能付き: 書籍名・借り手・貸出日）
- **FR-14**: 貸出履歴の閲覧
- **FR-22**: 貸出一覧の借り手による絞り込み機能

### 2.5 ユーザー認証・管理
- **FR-15**: ID/パスワードによるログイン認証（アプリ独自のユーザー管理）
- **FR-16**: ユーザー登録（管理者が新規ユーザーを作成）
- **FR-17**: 全ユーザーが同一権限（書籍登録・編集・貸出管理すべて可能）
- **FR-18**: ログアウト機能
- **FR-19**: パスワードのハッシュ化保存（bcrypt）
- **FR-20**: JWT（JSON Web Token）によるセッション管理

### 2.6 データ移行
- **FR-21**: 既存のExcelファイルからの書籍データ一括インポート（CSV / Excel対応）

---

## 3. 非機能要件

### 3.1 パフォーマンス
- **NFR-01**: 書籍一覧・検索結果は2秒以内に表示
- **NFR-02**: 書籍数50冊以下・同時利用者10名以下での動作を保証

### 3.2 可用性
- **NFR-03**: 社内営業時間内（平日 9:00〜18:00）の稼働を目標
- **NFR-04**: 計画外ダウンタイムは許容範囲内（社内ツールのため厳密なSLAは不要）

### 3.3 セキュリティ
- **NFR-05**: 社内LAN内のみでアクセス可能（インターネット公開なし）
- **NFR-06**: パスワードはbcryptでハッシュ化して保存（平文保存禁止）
- **NFR-07**: JWTトークンによる認証。有効期限を設定する
- **NFR-08**: 基本的なSQLインジェクション・XSS対策を実装（ORMのパラメータバインドを使用）
- **NFR-09**: APIエンドポイントは認証トークンのある場合のみアクセス可能（ログイン・ヘルスチェック以外）

### 3.4 保守性
- **NFR-10**: コードはモジュール化し、将来の機能追加が容易な構造とする
- **NFR-11**: APIはRESTful設計に準拠し、将来的なフロントエンド変更に対応

### 3.5 デプロイ・インフラ
- **NFR-12**: アプリケーション全体をDockerコンテナで稼働させる
- **NFR-13**: `docker-compose` で一括起動できる構成（フロントエンド + バックエンド + DB）
- **NFR-14**: 環境は1種類のみ（本番・開発の切り替えなし）

### 3.6 データ
- **NFR-15**: データはPostgreSQLに保存（単一環境）
- **NFR-16**: Excelインポート機能でデータ移行をサポート

## 4. 技術スタック

| レイヤー | 技術選択 | 理由 |
|---|---|---|
| バックエンド | Python 3.12 + FastAPI | ユーザー指定。高速・軽量・型安全なREST APIフレームワーク |
| 認証 | JWT（python-jose） + bcrypt（passlib） | シンプルなトークンベース認証。Pythonエコシステムの標準的な実装 |
| フロントエンド | React 18 + TypeScript | モダンなSPA。コンポーネント設計で保守性が高い |
| UIライブラリ | Tailwind CSS + shadcn/ui | 開発速度が速く、モダンなUI |
| データベース | PostgreSQL 16 | ユーザー指定。信頼性の高いRDB |
| ORM | SQLAlchemy 2.x + Alembic | Pythonの標準的なORM。マイグレーション管理も可能 |
| コンテナ | Docker + Docker Compose | ユーザー指定。環境の再現性を確保 |

## 6. データモデル（概要）

### ユーザー (User)
- id, username, hashed_password, display_name
- created_at

### 書籍 (Book)
- id, title, author, isbn, publisher, published_year
- category_id, location_id（FK: Location）
- created_at, updated_at

### カテゴリ (Category)
- id, name, description

### 保管場所 (Location)
- id, name

### 貸出記録 (LoanRecord)
- id, book_id, user_id（FK: User）, loan_date, return_date（NULL=貸出中）

## 7. スコープ外（今回は含めない）

- 貸出予約機能
- バーコードスキャン機能
- 通知機能（メール・Slack等）
- クラウドデプロイ
- モバイルアプリ対応
- ロールベースのアクセス制御（管理者/一般の権限分け）

---

## 8. 拡張機能設定

| Extension | 有効 | 理由 |
|---|---|---|
| Security Baseline | No | 社内ツール・最小限のセキュリティでよいとのユーザー判断（認証の基本実装は要件として別途含める） |
| Resiliency Baseline | No | 小規模・社内ツールのためスキップ |
| Property-Based Testing | No | シンプルなCRUDアプリのためスキップ |

---

## 9. 成功基準

- 部署メンバーがログインして書籍を検索・閲覧できる
- 貸出・返却の状況がリアルタイムで確認できる
- 既存ExcelデータをCSV/Excel経由でインポートできる
- 書籍の保管場所が登録・検索できる
- Docker Composeで `docker compose up` 一発で起動できる
